# 法眼识契 — 合同风险智能把控系统

## 一、系统总览

### 1.1 项目定位
面向企业法务/商务人员的智能合同审查工具，集成 OCR 识别 + 法律法规库 + 判例数据库 + AI 风险分析，实现"拍照→识别→分析→建议"全流程。

### 1.2 核心能力矩阵

| 维度 | 能力 | 技术实现 |
|------|------|----------|
| 输入层 | 拍照/相册上传 | 微信 wx.chooseImage / 百度OCR |
| 识别层 | 合同文本提取 | 百度OCR通用文字识别 / PaddleOCR |
| 知识层 | 法律法规检索 | 民法典合同编、劳动合同法、公司法等结构化库 |
| 判例层 | 相似案例匹配 | 裁判文书网结构化 + 向量检索 |
| 分析层 | 风险智能识别 | DeepSeek大模型 + 风险规则引擎 |
| 输出层 | 风险报告生成 | 结构化报告 + PDF导出 |

### 3.2 数据结构

```sql
-- 法律法规表
CREATE TABLE laws (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,           -- 法律名称
    category VARCHAR(50) NOT NULL,          -- 分类
    level ENUM('法律','行政法规','司法解释','地方法规','部门规章'),
    publish_date DATE,                      -- 公布日期
    effective_date DATE,                    -- 生效日期
    status ENUM('有效','已被修改','已被废止'),
    full_text LONGTEXT,                     -- 全文
    structured_json JSON,                   -- 结构化条款（章节→条→款→项）
    embedding VECTOR(1536),                 -- 语义向量（用于AI检索）
    updated_at TIMESTAMP
);

-- 条款级索引
CREATE TABLE law_articles (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    law_id BIGINT NOT NULL,
    chapter VARCHAR(100),                   -- 章
    section VARCHAR(100),                   -- 节
    article_no VARCHAR(20) NOT NULL,        -- 第X条
    content TEXT NOT NULL,                  -- 条款内容
    keywords TEXT,                          -- 关键词（全文检索用）
    embedding VECTOR(1536),
    FOREIGN KEY (law_id) REFERENCES laws(id)
);
```

## 四、判例库设计方案

### 4.1 数据来源

| 来源 | 覆盖范围 | 更新频率 |
|------|----------|----------|
| 中国裁判文书网 | 全国法院裁判文书 | 每日 |
| 最高人民法院公报案例 | 指导性案例 | 随发布 |
| 各地高院参考性案例 | 省级典型案例 | 每季度 |
| 北大法宝/威科先行 | 精选商事案例 | 付费订阅 |

### 4.2 判例数据结构

```json
{
  "case_id": "(2024)京01民终12345号",
  "court_level": "中级人民法院",
  "court_name": "北京市第一中级人民法院",
  "case_type": "民事",
  "cause": "买卖合同纠纷",
  "parties": {
    "plaintiff": "原告名称",
    "defendant": "被告名称"
  },
  "judge_date": "2024-03-15",
  "key_issues": ["合同效力认定", "违约金过高调整"],
  "legal_basis": ["民法典第585条", "民法典合同编司法解释第X条"],
  "judgment_summary": "判决要旨...",
  "contract_clauses": {
    "disputed_clauses": ["第X条 违约责任"],
    "dispute_focus": "违约金超过实际损失30%"
  },
  "verdict": "驳回上诉，维持原判",
  "embedding": [0.123, ...],   // 语义向量
  "tags": ["违约金", "买卖合同", "格式条款"]
}
```

### 4.3 判例匹配算法

```
输入：合同风险点 + 合同类型
  ↓
① 关键词匹配（文书类型 × 案由 × 争议条款类型）
  ↓
② 语义向量检索（FAISS/Milvus → top-50）
  ↓
③ 相关性排序（BM25 + 语义得分加权）
  ↓
④ 去重（同一案件多审判决合并）
  ↓
输出：Top-10 关联判例（含裁判要旨+适用法条）
```

## 五、合同风险分析引擎

### 5.1 分析流程

```
用户上传合同照片
    │
    ▼
┌──────────────┐
│   OCR识别     │ ← 百度OCR / PaddleOCR
│   + 人工校对  │ ← 用户可编辑修正识别错误
└──────┬───────┘
       │
       ▼
┌──────────────────┐
│  合同类型识别      │ ← LLM判断：买卖合同/劳动合同/租赁合同…
└──────┬───────────┘
       │
       ▼
┌────────────────────────────┐
│  逐条风险扫描               │ ← 并行执行
│  ├─ 主体资格风险            │   核查签约方资质
│  ├─ 标的物风险              │   规格/质量/交付标准
│  ├─ 价款/支付风险           │   金额/方式/时间/发票
│  ├─ 违约责任风险            │   违约情形/违约金比例
│  ├─ 争议解决风险            │   管辖/仲裁/法律适用
│  ├─ 保密/知识产权风险       │   归属/侵权责任
│  ├─ 格式条款风险            │   不合理免除/加重责任
│  ├─ 合同效力风险            │   无效/可撤销/效力待定
│  ├─ 担保/保证风险           │   保证方式/期间
│  └─ 程序风险               │   诉讼时效/管辖
└──────┬────────────────────┘
       │
       ▼
┌──────────────────────┐
│  关联法律条款 + 判例   │ ← 混合检索
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│  风险分级与评分         │
│  🔴 高风险 (8-10分)    │ ← 可能导致合同无效/重大损失
│  🟡 中风险 (4-7分)     │ ← 显著不利条款
│  🟢 低风险 (1-3分)     │ ← 可接受/建议优化
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│  生成专业风险报告       │ ← PDF/HTML
│  含：风险总览           │
│     逐条风险分析        │
│     法律依据            │
│     关联判例            │
│     修改建议            │
│     谈判策略建议        │
└──────────────────────┘
```

### 5.2 风险分析 Prompt 模板

```
你是一位资深中国商业法律师。请分析以下合同条款中的法律风险：

【合同全文】
{contract_text}

【合同类型】
{contract_type}

【分析维度】
1. 主体资格风险 - 签约方是否适格、有无资质要求
2. 标的条款风险 - 规格/质量标准/交付是否明确
3. 价款支付风险 - 金额/币种/支付节点/发票合规
4. 违约责任风险 - 违约金比例是否过高(>30%)、归责原则
5. 争议解决风险 - 管辖条款效力、仲裁约定明确性
6. 格式条款风险 - 是否存在不公平免除/加重责任
7. 合同效力风险 - 是否违反强制性规定、是否需要批准
8. 担保条款风险 - 保证方式/期间约定是否明确
9. 时效与程序风险 - 诉讼时效、通知期限、异议期

【输出格式】
每条风险输出：
- 风险等级：🔴高风险/🟡中风险/🟢低风险
- 风险描述：具体条款位置 + 法律问题
- 法律依据：引用具体法律条文
- 关联判例：相似裁判案例要旨
- 修改建议：具体修改措辞方案
- 谈判策略：底线条款 vs 可妥协条款
```

## 十、合规与数据安全

1. **合同内容保密**：所有合同数据加密存储，不用于模型训练
2. **法律建议免责声明**：明确标注"AI生成建议仅供参考，不构成正式法律意见"

4. **用户授权**：OCR和AI分析需用户明确同意
